# 🎯 @查询功能使用指南

## 功能概述

@查询功能允许您明确指定某些查询应该通过MCP（Model Context Protocol）服务处理，而不是使用数据库插件或其他默认处理方式。

## 语法格式

```
@<查询类型> <查询内容>
```

## 支持的查询类型

### 1. @help - 显示帮助信息
```
@help
```
显示所有支持的@查询类型和使用示例。

### 2. @web - 网页搜索和内容查询
```
@web python教程
@web https://example.com
```
- 支持关键词搜索网页内容
- 支持直接访问URL获取内容

### 3. @code - 编程代码示例查询
```
@code 机器学习算法
@code python函数
```
提供编程相关的代码示例和最佳实践。

### 4. @api - API相关信息查询
```
@api RESTful设计
@api 认证方式
```
获取API设计规范、认证方式等信息。

### 5. @data - 数据分析相关查询
```
@data 数据可视化
@data 机器学习
```
提供数据分析建议和常用工具信息。

### 6. @train - 列车时刻表查询
```
@train G902
@train 高铁902次
```
查询列车时刻表和运行信息。

### 7. 通用@查询
```
@人工智能发展趋势
@任何您想通过MCP查询的内容
```
对于没有特定类型前缀的@查询，系统会进行通用处理。

## 使用示例

### 基本查询
```
@web python教程          # 网页搜索Python教程
@code 排序算法           # 获取排序算法代码示例
@api GraphQL             # 查询GraphQL API信息
@data 数据清洗           # 获取数据清洗建议
@train D123              # 查询D123次列车时刻表
```

### 高级查询
```
@web https://github.com/microsoft/vscode  # 直接访问GitHub页面
@code tensorflow模型训练                 # 获取TensorFlow相关代码
@api OAuth2认证流程                      # 查询OAuth2认证规范
```

## 功能特点

### ✅ 智能识别
- 自动识别@开头的查询
- 明确标记为MCP查询类型
- 避免与数据库查询混淆

### ✅ 多样化支持
- 支持多种查询类型
- 每个类型都有专门的处理逻辑
- 提供丰富的示例和建议

### ✅ 缓存机制
- 查询结果自动缓存
- 提高响应速度
- 减少重复查询

### ✅ 兼容性
- 不影响现有功能
- 普通查询仍然正常工作
- 插件系统保持兼容

## 注意事项

1. **@符号必须在行首**：查询必须以@开头
2. **空格分隔**：@和查询内容之间用空格分隔
3. **类型前缀**：建议使用特定类型前缀获得更好结果
4. **缓存生效**：相同查询会使用缓存结果

## 故障排除

### 问题：查询没有被识别为@查询
**解决**：确保@符号在行首，后面有空格

### 问题：返回数据库结果而不是MCP结果
**解决**：检查查询是否正确以@开头

### 问题：@查询响应时间较长
**解决**：这是正常现象，因为需要调用MCP服务

## 技术实现

- **任务分析**：自动识别@查询并标记类型
- **路由控制**：@查询跳过插件处理，直接使用MCP
- **服务调用**：根据查询类型调用相应的MCP处理方法
- **结果缓存**：查询结果存储在本地数据库中
